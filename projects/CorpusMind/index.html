CorpusMind: From Raw Texts and Images to Publication-Ready Analysis

كوربَس مايند: من النصوص والصور الخام إلى تحليل جاهز للنشر

A local-first, AI-native research environment that enables linguists to conduct reproducible corpus and multimodal discourse analysis without writing code, without transmitting unpublished data to third-party servers, and without compromising methodological transparency.
بيئة بحثية محلية أولاً ومدعومة بالذكاء الاصطناعي تُمكّن اللغويين من إجراء تحليلات لغوية وخطابية متعددة الوسائط قابلة للتكرار دون كتابة سطر برمجي واحد، ودون إرسال البيانات غير المنشورة إلى خوادم خارجية، ودون التضحية بالشفافية المنهجية التي يفرضها التحكيم العلمي.
Local-First Windows macOS Linux Ollama / LM Studio AGPL-3.0-only

CorpusMind is an integrated research environment that unifies corpus linguistics, multimodal discourse analysis, and AI-assisted interpretation within a single desktop application. The platform comprises three coordinated components: a FastAPI engine (Python 3.12) that performs natural language processing, statistical analysis, and computer vision inference; a Tauri 2 shell (Rust) that supervises the engine lifecycle and enforces a hardened security boundary between the webview and the host system; and an installable Progressive Web App (React 18, Vite, TypeScript) that provides the analyst interface. The system supports English and Arabic text analysis through spaCy and CAMeL Tools, offers twelve discourse-analysis frameworks grounded in the peer-reviewed literature, and integrates a pluggable AI assistant capable of connecting to local model providers (Ollama, LM Studio) or, at the user's explicit election, remote providers (OpenAI, Anthropic). All analytical operations produce YAML provenance records, ensuring that every result is traceable, reproducible, and amenable to peer-review scrutiny.

كوربَس مايند بيئة بحثية متكاملة توحّد تحليل اللغة والخطاب متعدد الوسائط والتفسير المدعوم بالذكاء الاصطناعي ضمن تطبيق مكتبي واحد. تتكوّن المنصة من ثلاثة مكونات متناسقة: محرك FastAPI (Python 3.12) ينفّذ معالجة اللغة الطبيعية والتحليل الإحصائي والاستدلال البصري؛ وقشرة Tauri 2 (Rust) تشرف على دورة حياة المحرك وتفرض حدوداً أمنية صارمة بين واجهة الويب والنظام المضيف؛ وتطبيق ويب تقدمي قابل للتثبيت (React 18، Vite، TypeScript) يوفّر واجهة المحلل. يدعم النظام تحليل النصوص الإنجليزية والعربية عبر spaCy وCAMeL Tools، ويقدّم اثني عشر إطاراً لتحليل الخطاب مستندة إلى الأدبيات المحكّمة، ويدمج مساعداً ذكياً اصطناعياً قابلاً للتوصيل يدعم مزوّدي النماذج المحليين (Ollama، LM Studio) أو، باختيار صريح من المستخدم، المزوّدين البعيدين (OpenAI، Anthropic). تُنتج جميع العمليات التحليلية سجلات إثبات بصيغة YAML، مما يضمن إمكانية تتبع كل نتيجة وإعادة إنتاجها وخضوعها للتدقيق العلمي.

Research Integrity Statement

إقرار نزاهة البحث

CorpusMind is designed for researchers who require methodologically transparent computational analysis of linguistic and multimodal data. The platform enforces a strict provenance contract: every analytical output is accompanied by a YAML record specifying the corpus, the pipeline configuration, the statistical measure, the software version, and the timestamp. The AI assistant operates under a citation-enforced contract — it must either cite the corpus evidence supporting its claim or explicitly flag the claim as unsupported. The system does not fabricate evidence, does not permit the suppression of analytical steps, and does not transmit user data to any remote server unless the user explicitly configures a remote model provider.

صُمِّم كوربَس مايند للباحثين الذين يتطلبون شفافية منهجية في التحليل الحسابي للبيانات اللغوية ومتعددة الوسائط. تفرض المنصة عقد إثبات صارم: كل مُخرج تحليلي يصحبه سجل YAML يحدّد المجموعة اللغوية وإعداد خط المعالجة والمقياس الإحصائي وإصدار البرنامج والطابع الزمني. يعمل المساعد الذكي بموجب عقد إلزامي بالاستشهاد — عليه إما الاستشهاد بأدلة المجموعة اللغوية الداعمة لادعائه أو الإشارة الصريحة إلى أن الادعاء غير مدعوم. لا يختلق النظام أدلة، ولا يسمح بإخفاء الخطوات التحليلية، ولا ينقل بيانات المستخدم إلى أي خادم بعيد ما لم يُعدّ المستخدم صراحةً مزوّداً بعيداً للنماذج.

Design Pillars

ركائز التصميم

Local-First Architecture

معمارية محلية أولاً

All computational operations — natural language processing, statistical analysis, vision inference, and AI-assisted interpretation — execute on the researcher's own hardware. The default configuration permits no outbound network traffic. The Content Security Policy in tauri.conf.json restricts the webview to 127.0.0.1:8765 (the local engine), 127.0.0.1:11434 (Ollama), and 127.0.0.1:1234 (LM Studio). A remote model provider (OpenAI, Anthropic) may be configured only through an explicit opt-in action by the user, and the system discloses the data-transmission implications of that choice before the configuration is saved.

جميع العمليات الحسابية — معالجة اللغة الطبيعية والتحليل الإحصائي والاستدلال البصري والتفسير المدعوم بالذكاء الاصطناعي — تُنفَّذ على عتاد الباحث نفسه. الإعداد الافتراضي لا يسمح بأي اتصال شبكي خارجي. تقيّد سياسة أمان المحتوى في tauri.conf.json واجهة الويب بـ 127.0.0.1:8765 (المحرك المحلي)، و127.0.0.1:11434 (Ollama)، و127.0.0.1:1234 (LM Studio). يمكن إعداد مزوّد نماذج بعيد (OpenAI، Anthropic) فقط من خلال إجراء اشتراك صريح من المستخدم، ويكشف النظام عن تبعات نقل البيانات لهذا الخيار قبل حفظ الإعداد.

Methodological Transparency

الشفافية المنهجية

Every analytical operation emits a YAML provenance record specifying the corpus identifier, the document selection criteria, the pipeline recipe (tokenizer, tagger, parser, NER model), the statistical measure with its parameters (window size, significance threshold, smoothing), the software version, and the timestamp. These records are exportable alongside the analytical results and are designed to satisfy the reproducibility requirements of peer-reviewed venues in linguistics, discourse analysis, and computational social science. The methodology reference manual documents the formal definition of every implemented measure, with citations to the primary literature.

كل عملية تحليلية تُصدر سجل إثبات بصيغة YAML يحدّد معرّف المجموعة اللغوية ومعايير اختيار الوثائق ووصفة خط المعالجة (المُرمِّز، والوسم، والمحلل، ونموذج التعرف على الكيانات) والمقياس الإحصائي مع معاملاته (حجم النافذة، عتبة الدلالة، التنعيم) وإصدار البرنامج والطابع الزمني. هذه السجلات قابلة للتصدير مع النتائج التحليلية، وهي مصممة لتلبية متطلبات إعادة الإنتاج في المجلات المحكّمة في اللسانيات وتحليل الخطاب والعلوم الاجتماعية الحسابية. يوثّق دليل المنهجية التعريف الرسمي لكل مقياس مُنفَّذ، مع الاستشهاد بالأدبيات الأولية.

Framework-Grounded Analysis

تحليل قائم على الأطر النظرية

CorpusMind operationalizes twelve established frameworks from the discourse-analysis and multimodal-analysis literature: Halliday's Systemic Functional Linguistics; Kress and van Leeuwen's Visual Grammar; Barthesian semiotics; Peircean semiotics; Fairclough's Critical Discourse Analysis; Wodak's Discourse-Historical Approach; van Dijk's Socio-Cognitive Approach; Machin and Mayr's Multimodal Critical Discourse Analysis; Martin and White's Appraisal Theory; Lakoff and Johnson's Conceptual Metaphor Theory; Toulmin's argumentation model; and Aristotle's rhetorical framework. Each framework is implemented as a structured annotation template with a YAML schema, ensuring that the analyst's interpretive decisions are recorded in a machine-readable form amenable to inter-coder reliability testing.

يُفعّل كوربَس مايند اثني عشر إطاراً راسخاً من أدبيات تحليل الخطاب والتحليل متعدد الوسائط: اللسانيات الوظيفية النظامية لهاليداي؛ وقواعد الصورة المرئية لكريس وفان ليوفان؛ وسيميائية بارت؛ وسيميائية بيرس؛ والتحليل النقدي للخطاب لفيركلو؛ والمنهج التاريخي للخطاب لفوداك؛ والمنهج الاجتماعي المعرفي لفان دايك؛ والتحليل النقدي متعدد الوسائط لماتشين وماير؛ ونظرية التقييم لمارتن ووايت؛ ونظرية الاستعارة المفاهيمية للاكوف وجونسون؛ ونموذج تولمين للحجاج؛ والإطار البلاغي لأرسطو. كل إطار مُنفَّذ كقالب شرح مهيكل بمخطط YAML، مما يضمن تسجيل قرارات المحلل التفسيرية بصيغة قابلة للقراءة آلياً وقابلة لاختبار موثوقية المُرمِّزين.

Core Capabilities

القدرات الأساسية

Concordancer with Stable Line Identifiers

الكشف السياقي بمعرّفات أسطر ثابتة

Key-Word-In-Context (KWIC) search with stable line identifiers that the AI assistant cites when grounding its claims. Supports lemma, word, and POS levels; wildcard queries; regex patterns; and filterable context windows. Every concordance line is addressable by a permanent identifier, enabling the analyst and the AI assistant to refer to the same evidence unambiguously across the research lifecycle.

بحث الكلمة السياقية (KWIC) بمعرّفات أسطر ثابتة يستشهد بها المساعد الذكي عند تأسيس ادعاءاته. يدعم مستويات اللفظ والكلمة والفئة النحوية؛ والاستعلامات بالبدل؛ وأنماط التعبيرات النمطية؛ والنوافذ السياقية القابلة للتصفية. كل سطر سياقي قابل للإشارة بمعرّف دائم، مما يُمكّن المحلل والمساعد الذكي من الإشارة إلى الدليل نفسه دون لبس عبر دورة حياة البحث.

Collocation and Keyness Statistics

إحصاءات التضايق والتمييز

Implements all seven collocation measures documented in the methodology literature: Mutual Information, T-score, Log-Likelihood, Dice, Log-Dice, Chi-Square, and ΔP. Keyness analysis supports Log-Likelihood Ratio and Burrows's Zeta. Every measure is parameterized (window size, frequency threshold, reference corpus) and emits a provenance record. The implementation has been validated against the published formulas and tested with 20 unit tests covering edge cases.

ينفّذ جميع مقاييس التضايق السبعة الموثقة في أدبيات المنهجية: المعلومات المتبادلة، وT-score، والاحتمالية اللوغاريتمية، وDice، وLog-Dice، ومربع كاي، وΔP. يدعم تحليل التمييز نسبة الاحتمالية اللوغاريتمية وZeta لباروز. كل مقياس مُعامل (حجم النافذة، عتبة التردد، المجموعة المرجعية) ويُصدر سجل إثبات. تم التحقق من التنفيذ مقابل الصيغ المنشورة واختباره بـ 20 اختبار وحدة يغطي الحالات الحرجة.

English and Arabic NLP Pipelines

خطوط معالجة اللغة الإنجليزية والعربية

The English pipeline uses spaCy (en_core_web_sm) for tokenization, lemmatization, POS tagging, dependency parsing, and named-entity recognition. The Arabic pipeline integrates CAMeL Tools for morphology, dialect identification, named-entity recognition, and sentiment analysis; SinaTools for tokenization and lemmatization; and Farasa for segmenting Arabic text. Bilingual alignment is supported for parallel-corpus analysis. The system documents which pipeline was applied to which document, ensuring that English and Arabic results are not confounded in cross-lingual comparisons.

يستخدم خط المعالجة الإنجليزي spaCy (en_core_web_sm) للترميز والتأصيل الصرفي والوسم النحوي والتحليل التبعي والتعرف على الكيانات المسماة. يدمج خط المعالجة العربي CAMeL Tools للصرف وتحديد اللهجة والتعرف على الكيانات وتحليل المشاعر؛ وSinaTools للترميز والتأصيل؛ وFarasa لتجزئة النص العربي. يُدعم المحاذاة ثنائية اللغة لتحليل المجموعات المتوازية. يوثّق النظام أي خط طُبّق على أي وثيقة، مما يضمن عدم خلط النتائج الإنجليزية والعربية في المقارنات بين اللغات.

Multimodal Vision Pipeline

خط المعالجة البصرية متعدد الوسائط

Computer-vision analysis of still images using OpenCV and Pillow: object detection, facial analysis (age, gender, emotion estimates with confidence intervals), OCR via Tesseract (with Arabic language pack support), color palette extraction, composition analysis (rule of thirds, symmetry, salience), and metadata extraction. Visual-Grammar annotation operationalizes Kress and van Leeuwen's representational, interactive, and compositional metafunctions. Multimodal alignment links visual elements to co-occurring textual segments for integrated discourse analysis.

تحليل رؤية حاسوبية للصور الثابتة باستخدام OpenCV وPillow: كشف الكائنات، وتحليل الوجوه (تقديرات العمر والجنس والعاطفة بفترات ثقة)، والتعرف الضوئي على الحروف عبر Tesseract (مع دعم حزمة اللغة العربية)، واستخراج لوحة الألوان، وتحليل التكوين (قاعدة الأثلاث، التناظر، البروز)، واستخراج البيانات الوصفية. شرح قواعد الصورة المرئية يُفعّل الوظائف التمثيلية والتفاعلية والتركيبية لكريس وفان ليوفان. المحاذاة متعددة الوسائط تربط العناصر البصرية بالأجزاء النصية المتزامنة لتحليل خطابي متكامل.

Grounded AI Assistant

المساعد الذكي المؤسَّس

A tool-calling agent with twenty-five registered tools that query the corpus, the statistical results, and the vision pipeline. The assistant operates under a citation-enforced contract: every factual claim must be supported by an explicit reference to a concordance line, a collocation result, or a vision annotation. Claims lacking supporting evidence are flagged as unsupported rather than suppressed. The assistant can connect to local model providers (Ollama, LM Studio) for fully offline operation, or to remote providers (OpenAI, Anthropic, Google) when configured by the user. The system discloses which provider processed each query in the provenance record.

عميل استدعاء أدوات بخمسة وعشرين أداة مسجّلة تستعلم المجموعة اللغوية والنتائج الإحصائية وخط المعالجة البصرية. يعمل المساعد بموجب عقد إلزامي بالاستشهاد: كل ادعاء وقائعي يجب أن يدعمه إشارة صريحة إلى سطر سياقي أو نتيجة تضايق أو شرح بصري. الادعاءات التي تفتقر إلى الأدلة الداعمة تُعلَّم كـ غير مدعومة بدلاً من إخفائها. يمكن للمساعد الاتصال بمزوّدي النماذج المحليين (Ollama، LM Studio) لعمل غير متصل بالكامل، أو بالمزوّدين البعيدين (OpenAI، Anthropic، Google) عند إعدادهم من قبل المستخدم. يكشف النظام أي مزوّد عالج كل استعلم في سجل الإثبات.

Publication-Ready Export

تصدير جاهز للنشر

Export analytical results to Excel (.xlsx) for tabular data, Word (.docx) for narrative reports, PDF for distribution, and LaTeX for manuscript integration. Every export includes the provenance YAML inline, enabling reviewers to verify the analytical chain. The Word and LaTeX exports follow the formatting conventions of linguistic journals (e.g., table captions, KWIC line numbering, statistical notation). A methods.pdf export generates a human-readable methods section documenting every analytical step applied to the corpus, suitable for inclusion in a manuscript's methodology section.

صدّر النتائج التحليلية إلى Excel (.xlsx) للبيانات الجدولية، وWord (.docx) للتقارير السردية، وPDF للتوزيع، وLaTeX للدمج في المخطوطات. كل تصدير يتضمن سجل إثبات YAML مُضمَّناً، مما يُمكّن المحكّمين من التحقق من السلسلة التحليلية. تصديرات Word وLaTeX تتبع اصطلاحات تنسيق المجلات اللسانية (مثل تعليقات الجداول، ترقيم أسطر KWIC، التدوين الإحصائي). تصدير methods.pdf يُنشئ قسماً منهجياً مقروءاً بشرياً يوثّق كل خطوة تحليلية طُبقت على المجموعة اللغوية، مناسب للإدراج في قسم المنهجية في المخطوطة.

At-Rest Encryption for Sensitive Corpora

تشفير البيانات الساكنة للمجموعات الحساسة

Optional at-rest encryption for corpora containing sensitive or unpublished data. When enabled, document contents are encrypted using AES-256 with a user-supplied passphrase; the passphrase is never persisted to disk and is required on each session. The encryption key is derived using PBKDF2 with 600,000 iterations. The SQLite metadata database, the search index, and the cached analytical results are all encrypted. This feature supports compliance with institutional data-governance policies and funder mandates (e.g., GDPR Article 32, NSF Data Management Plans).

تشفير اختياري للبيانات الساكنة للمجموعات التي تحتوي على بيانات حساسة أو غير منشورة. عند التفعيل، تُشفَّر محتويات الوثائق باستخدام AES-256 بعبارة مرور يزوّدها المستخدم؛ لا تُحفظ عبارة المرور على القرص أبداً وتُطلب في كل جلسة. يُشتق مفتاح التشفير باستخدام PBKDF2 بـ 600,000 تكرار. تُشفَّر قاعدة بيانات SQLite للبيانات الوصفية وفهرس البحث والنتائج التحليلية المُخزَّنة. تدعم هذه الميزة الامتثال لسياسات حوكمة البيانات المؤسسية ومتطلبات المموّلين (مثل المادة 32 من GDPR، وخطط إدارة بيانات NSF).

Reproducibility and Provenance

إعادة الإنتاج وسجل الإثبات

Every analytical operation — from corpus ingestion through export — is recorded in a YAML provenance file that captures the operation type, the input corpus, the pipeline configuration, the parameters, the software version, the timestamp, and the model provider (if applicable). These records are exportable, machine-readable, and designed to satisfy the reproducibility requirements of peer-reviewed venues. A methods.pdf export compiles all provenance records for a given corpus into a single narrative methods section, suitable for inclusion in a manuscript.

كل عملية تحليلية — من استيعاب المجموعة اللغوية إلى التصدير — مُسجَّلة في ملف إثبات YAML يلتقط نوع العملية والمجموعة الداخلة وإعداد الخط والمعاملات وإصدار البرنامج والطابع الزمني ومزوّد النموذج (إن وُجد). هذه السجلات قابلة للتصدير ومقروءة آلياً ومصممة لتلبية متطلبات إعادة الإنتاج في المجلات المحكّمة. تصدير methods.pdf يجمع جميع سجلات الإثبات لمجموعة لغوية معينة في قسم منهجي سردي واحد، مناسب للإدراج في مخطوطة.

Implemented Theoretical Frameworks

الأطر النظرية المُنفَّذة

Systemic Functional Linguistics

Halliday & Matthiessen (2014). Three metafunctions: ideational, interpersonal, textual.

Visual Grammar

Kress & van Leeuwen (2006). Reading Images: representational, interactive, compositional.

Barthesian Semiotics

Barthes (1964/1977). Denotation, connotation, myth, anchorage, relay.

Peircean Semiotics

Peirce (1931-1958). Sign, object, interpretant; icon, index, symbol.

Critical Discourse Analysis

Fairclough (1995). Three-dimensional framework: text, discursive practice, social practice.

Discourse-Historical Approach

Wodak & Meyer (2009). Four levels of context: co-text, intertextual, interdiscursive, broad socio-political.

Socio-Cognitive Approach

van Dijk (2008). Cognitive, social, and discursive dimensions of ideology reproduction.

Multimodal Critical Discourse Analysis

Machin & Mayr (2012). Combines CDA with visual-grammar analysis of ideological content.

Appraisal Theory

Martin & White (2005). Attitude, engagement, graduation in evaluative language.

Conceptual Metaphor Theory

Lakoff & Johnson (1980). Source-target domain mappings and entailments.

Toulmin Argumentation

Toulmin (1958). Claim, data, warrant, backing, qualifier, rebuttal.

Aristotelian Rhetoric

Aristotle (c. 350 BCE). Ethos, pathos, logos; the rhetorical situation.

A remote provider is contacted only when the user explicitly configures one. الاتصالات الشبكية الخارجية لا شيء افتراضياً. يرتبط المحرك بـ 127.0.0.1:8765؛ ويرتبط المساعد الذكي بـ 127.0.0.1:11434 (Ollama) أو 127.0.0.1:1234 (LM Studio). يُتصل بمزوّد بعيد فقط عندما يُعدّه المستخدم صراحةً.
Corpus contents Never leave the device unless a remote model provider is explicitly configured. Even then, only the prompts sent to that provider are transmitted; the corpus itself remains on the device. محتويات المجموعة اللغوية لا تغادر الجهاز ما لم يُعدَّ مزوّد نماذج بعيد صراحةً. وحتى ذلك الحين، تُنقل فقط الاستفسارات المُرسلة إلى ذلك المزوّد؛ تبقى المجموعة اللغوية نفسها على الجهاز.
At-rest encryption Optional AES-256 with PBKDF2 key derivation (600,000 iterations). Passphrase never persisted. Covers SQLite metadata, search index, cached results. تشفير البيانات الساكنة اختياري AES-256 مع اشتقاق مفتاح PBKDF2 (600,000 تكرار). عبارة المرور لا تُحفظ أبداً. يغطي بيانات SQLite الوصفية وفهرس البحث والنتائج المُخزَّنة.
Provenance records YAML files stored locally alongside the corpus. Machine-readable, exportable, designed for peer-review scrutiny. سجلات الإثبات ملفات YAML محفوظة محلياً مع المجموعة اللغوية. مقروءة آلياً، قابلة للتصدير، مصممة للتدقيق العلمي.
AI assistant disclosures Each AI-assisted output records which provider processed the query (local or remote), which model was used, and whether the response was grounded in corpus evidence or flagged as unsupported. إفصاحات المساعد الذكي كل مُخرج مدعوم بالذكاء الاصطناعي يسجل أي مزوّد عالج الاستعلام (محلي أو بعيد)، وأي نموذج استُخدم، وما إذا كانت الاستجابة مؤسَّسة على أدلة المجموعة أو مُعلَّمة كغير مدعومة.
Third-party APIs None by default. OpenAI, Anthropic, and Google APIs are contacted only when the user explicitly configures an API key in Settings. واجهات الطرف الثالث لا شيء افتراضياً. يتم الاتصال بواجهات OpenAI وAnthropic وGoogle فقط عندما يُعدّ المستخدم مفتاح API صراحةً في الإعدادات.

What CorpusMind Does — and Does NOT Do

ما يفعله كوربَس مايند — وما لا يفعله

What it DOES

ما يفعله

  • Provides reproducible, framework-grounded analysis of linguistic and multimodal data.
  • يوفّر تحليلاً قابلاً لإعادة الإنتاج ومؤسَّساً على الأطر النظرية للبيانات اللغوية ومتعددة الوسائط.
  • Records every analytical operation in a machine-readable YAML provenance file.
  • يسجل كل عملية تحليلية في ملف إثبات YAML مقروء آلياً.
  • Enforces a citation contract on the AI assistant: every claim must cite corpus evidence or be flagged as unsupported.
  • يفرض عقد استشهاد على المساعد الذكي: كل ادعاء يجب أن يستشهد بأدلة المجموعة أو يُعلَّم كغير مدعوم.
  • Operates entirely on the researcher's own hardware when used with a local model provider.
  • يعمل بالكامل على عتاد الباحث عند استخدامه مع مزوّد نماذج محلي.
  • Supports at-rest encryption for corpora containing sensitive or unpublished data.
  • يدعم تشفير البيانات الساكنة للمجموعات التي تحتوي على بيانات حساسة أو غير منشورة.
  • Generates publication-ready exports with provenance metadata suitable for peer-review submission.
  • يُنشئ تصديرات جاهزة للنشر ببيانات إثبات وصفية مناسبة للتقديم للتحكيم.
  • Documents the formal definition and primary-literature citation for every implemented statistical measure.
  • يوثّق التعريف الرسمي والاستشهاد بالأدبيات الأولية لكل مقياس إحصائي مُنفَّذ.

What it explicitly does NOT do

ما لا يفعله صراحةً

  • Does not transmit corpus data to any remote server by default. The default configuration permits no outbound network traffic.
  • لا ينقل بيانات المجموعة إلى أي خادم بعيد افتراضياً. الإعداد الافتراضي لا يسمح بأي اتصال شبكي خارجي.
  • Does not fabricate evidence. The AI assistant must cite a concordance line, a collocation result, or a vision annotation, or explicitly flag the claim as unsupported.
  • لا يختلق أدلة. يجب على المساعد الذكي الاستشهاد بسطر سياقي أو نتيجة تضايق أو شرح بصري، أو الإشارة الصريحة إلى أن الادعاء غير مدعوم.
  • Does not suppress analytical steps. Every operation is recorded in the provenance file; no operation can be retroactively removed.
  • لا يخفي الخطوات التحليلية. كل عملية مُسجَّلة في ملف الإثبات؛ لا يمكن إزالة أي عملية بأثر رجعي.
  • Does not collect telemetry, analytics, or crash reports.
  • لا يجمع بيانات استخدام أو تحليلات أو تقارير أعطال.
  • Does not access any file the user has not explicitly imported through the file dialog.
  • لا يصل إلى أي ملف لم يُستورد صراحةً من خلال حوار الملفات.
  • Does not circumvent institutional data-governance policies. At-rest encryption and the local-only default configuration are designed to support compliance, not evade it.
  • لا يتهرب من سياسات حوكمة البيانات المؤسسية. تشفير البيانات الساكنة والإعداد المحلي الافتراضي مصممان لدعم الامتثال، لا التهرب منه.

Getting Started

البدء

1

Install CorpusMind

ثبّت كوربَس مايند

Download the latest installer for your platform from the GitHub Releases page: .dmg for macOS (Apple Silicon and Intel), .exe (NSIS) or .msi for Windows, .AppImage or .deb for Linux. Alternatively, build from source using the provided scripts/build-macos-arm64.sh script (macOS) or the GitHub Actions release workflow (all platforms).

نزّل أحدث مثبّت لمنصتك من صفحة الإصدارات على GitHub: .dmg لـ macOS (Apple Silicon وIntel)، .exe (NSIS) أو .msi لـ Windows، .AppImage أو .deb لـ Linux. بدلاً من ذلك، ابنِ من المصدر باستخدام السكربت المُقدَّم scripts/build-macos-arm64.sh (macOS) أو سير عمل إصدار GitHub Actions (كل المنصات).

2

(Optional) Install a Local Model Provider

(اختياري) ثبّت مزوّد نماذج محلي

To enable the AI assistant with full offline operation, install Ollama or LM Studio. With Ollama, pull a model suited to your hardware (e.g., ollama pull llama3.2:3b for 8 GB RAM, ollama pull qwen3:14b for 16 GB RAM). The AI assistant works with any OpenAI-compatible local endpoint.

لتفعيل المساعد الذكي بعمل غير متصل بالكامل، ثبّت Ollama أو LM Studio. مع Ollama، اسحب نموذجاً يناسب عتادك (مثل ollama pull llama3.2:3b لـ 8 جيجا رام، ollama pull qwen3:14b لـ 16 جيجا رام). يعمل المساعد الذكي مع أي نقطة نهاية محلية متوافقة مع OpenAI.

3

Create a Project and Import a Corpus

أنشئ مشروعاً واستورد مجموعة لغوية

Launch CorpusMind, create a project, and import documents in any of the supported formats: TXT, DOCX, PDF, HTML, XML, CSV, or Markdown. The ingestion pipeline performs charset detection, language identification, and full NLP processing (tokenization, lemmatization, POS tagging, dependency parsing, NER). The pipeline recipe is recorded in the provenance file for every document.

شغّل كوربَس مايند، أنشئ مشروعاً، واستورد الوثائق بأي من الصيغ المدعومة: TXT، DOCX، PDF، HTML، XML، CSV، أو Markdown. ينفّذ خط الاستيعاب كشف الترميز وتحديد اللغة ومعالجة اللغة الطبيعية الكاملة (الترميز، التأصيل الصرفي، الوسم النحوي، التحليل التبعي، التعرف على الكيانات). تُسجَّل وصفة الخط في ملف الإثبات لكل وثيقة.
