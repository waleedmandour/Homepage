ScholarScribe: Write with Ethical AI, On Your Own Machine

سكولار سكرايب: اكتب بذكاء اصطناعي أخلاقي على جهازك الخاص

A privacy-first, local-LLM writing companion for researchers — runs entirely on your device, no telemetry, no cloud calls, no paid APIs
رفيق كتابة بحثي يعتمد على النماذج اللغوية المحلية ويحترم الخصوصية أولاً — يعمل بالكامل على جهازك، بدون إرسال بيانات استخدام، وبلا اتصالات سحابية، وبدون رسوم
No Telemetry Windows macOS Linux Local LLMs (Ollama) Open Source (MIT)

ScholarScribe helps researchers who are writing their own manuscripts by running open large language models (Gemma 3, Qwen 3, Phi-4, DeepSeek R1, Llama 3.3) fully offline. It combines a 24-rule AI text cleaner, citation validation against BibTeX files, document statistics, structure analysis, abstract generation, authenticity-risk assessment, voice-consistency checks, a writing-process journal, AI-use disclosure statements, and an in-app privacy audit — all designed to support transparent, ethical research writing without ever sending your text to the cloud.

يساعد سكولار سكرايب الباحثين الذين يكتبون خططهم البحثية الخاصة من خلال تشغيل نماذج لغوية كبيرة مفتوحة (Gemma 3 وQwen 3 وPhi-4 وDeepSeek R1 وLlama 3.3) بالكامل دون اتصال. إذ يجمع التطبيق بين منظّف نصوص ذكي بقواعد التحول الـ 24، والتحقق من الاستشهادات مقابل ملفات BibTeX، وإحصاءات الوثيقة، وتحليل البنية، وتوليد الملخصات، وتقييم مخاطر الأصالة، وفحص اتساق الصوت، ومذكرات عملية الكتابة، وبيانات الإفصاح عن استخدام الذكاء الاصطناعي، وسجل خصوصية داخلي — كلها مصممة لدعم الكتابة البحثية الشفافة والأخلاقية دون إرسال نصك إلى السحابة أبداً.

Ethical Use — Please Read

الاستخدام الأخلاقي — يُرجى القراءة

ScholarScribe is designed for researchers who have genuinely written or substantially contributed to a manuscript and wish to work with AI assistance transparently and on their own device. It is not a tool for evading AI-detection systems, for misrepresenting AI-generated text as original human work, or for lowering Turnitin/GPTZero/Originality scores.

صُمِّم سكولار سكرايب للباحثين الذين كتبوا أو ساهموا فعلياً في مخطوطة ويرغبون في العمل مع الذكاء الاصطناعي بشفافية وعلى أجهزتهم الخاصة. وهو ليس أداةً للتهرب من أنظمة كشف الذكاء الاصطناعي، ولا لتمرير النصوص المولّدة بواسطة الذكاء الاصطناعي كعمل بشري أصيل، ولا لخفض درجات Turnitin أو GPTZero أو Originality.

Design Pillars

ركائز تصميم البرنامج

Local-First Privacy

الخصوصية علي جهازك أولاً

No telemetry, no analytics, no crash reports. The only outbound network call is to registry.ollama.ai when you choose to download a model — and that call carries no text, no prompts, no usage data. The CSP in tauri.conf.json explicitly restricts the UI to 127.0.0.1:11434 (your local Ollama). Every other outbound host is a red flag.

بدون إرسال بيانات استخدام، أو تحليلات، أو تقارير أعطال. الاتصال الخارجي الوحيد هو إلى registry.ollama.ai عند تنزيل نموذج — ولا يحمل هذا الطلب أي نص أو استفسارات أو بيانات استخدام. تقيّد سياسة أمان المحتوى (CSP) في tauri.conf.json واجهة المستخدم صراحةً على 127.0.0.1:11434 (Ollama المحلي). أي مضيف خارجي آخر يُعد إشارة خطر.

Research-Integrity First

نزاهة البحث أولاً

ScholarScribe validates your citations against your .bib file — listing undefined citations, unused references, and broken in-text citations — reducing the risk of fabricated references. The chat module's system prompt explicitly refuses requests to evade detectors or fabricate citations. The Citation Manager exists precisely to catch accidental fabrication by listing every in-text citation that doesn't match a real .bib entry.

يتحقق سكولار سكرايب من استشهاداتك مقابل ملف .bib الخاص بك — مُدرجاً الاستشهادات غير المعرفة، والمراجع غير المستخدمة، والاستشهادات النصية المعطوبة — مما يقلل من خطر المراجع المختلقة. ترفض رسالة النظام في وحدة المحادثة صراحةً طلبات التهرب من الكواشف أو اختلاق الاستشهادات. توجد إدارة الاستشهادات تحديداً لرصد الاختراق العرضي عبر سرد كل استشهاد نصي لا يطابق إدخالاً حقيقياً في ملف .bib.

Transparency & Disclosure

الشفافية والإفصاح

Generate venue-compliant AI-use disclosure statements for ICMJE, Nature, IEEE, Elsevier, ACL/EMNLP/NAACL, or any custom venue. Maintain a timestamped writing-process journal as verifiable evidence of authentic authorship. If falsely flagged by an AI detector, generate an evidence-based appeal letter citing the peer-reviewed literature (Liang et al. 2023; Weber-Wulff et al. 2023).

أنشئ بيانات إفصاح متوافقة مع معايير النشر حول استخدام الذكاء الاصطناعي لـ ICMJE وNature وIEEE وElsevier وACL/EMNLP/NAACL أو أي جهة ناشرة مخصصة. حافظ على مذكرات عملية كتابة مؤطرة زمنياً كدليل قابل للتحقق على الأصالة. وفي حال تم تصنيفك زوراً بواسطة كاشف ذكاء اصطناعي، أنشئ خطاب طعن قائم على الأدلة يستشهد بالأدبيات المحكمة (Liang et al. 2023؛ Weber-Wulff et al. 2023).

Core Features

الميزات الأساسية

Models Tab — Run Open LLMs Offline

تبويب النماذج — شغّل النماذج المفتوحة دون اتصال

Install and manage 14 open LLMs spanning 4–64 GB RAM: Gemma 3 (1B/4B/12B/27B), Qwen 3 (4B/8B/14B/32B), Phi-4 + Phi-4 Mini, DeepSeek R1 (14B/32B), Llama 3.3 70B, and SciGLM 6B. Import local .gguf files you downloaded from HuggingFace — ScholarScribe checks your RAM against the model's needs (file size × 1.5) and shows a verdict: ok, tight, or insufficient. GGUF imports make zero outbound network calls.

ثبّت وأدر 14 نموذجاً لغوياً مفتوحاً يتراوح بين 4 و64 جيجابايت رام: Gemma 3 (1B/4B/12B/27B)، Qwen 3 (4B/8B/14B/32B)، Phi-4 وPhi-4 Mini، DeepSeek R1 (14B/32B)، Llama 3.3 70B، وSciGLM 6B. استورد ملفات .gguf المحلية التي حمّلتها من HuggingFace — يتحقق سكولار سكرايب من رامك مقابل احتياجات النموذج (حجم الملف × 1.5) ويُظهر الحكم: ok، tight، أو insufficient. استيراد GGUF لا يؤدي إلى أي اتصال شبكي خارجي.

AI Text Cleaner — 24 Rule-Based Transformations

منظّف النص — 24 تحويلاً قاعدياً

Fix common artifacts from copy-pasted text — especially from PDFs, web pages, OCR, and word processors. Default (12 ops): fix mojibake, expand ligatures, normalize quotes/dashes, strip zero-width/control chars, join hyphenated line breaks, fix broken citations, remove page numbers, collapse whitespace. Strict (24 ops, ε button): adds BOM strip, CRLF→LF, non-breaking-space conversion, Unicode whitespace normalization, soft-hyphen strip, variation-selector strip, ellipsis conversion, asterisk removal, markdown-heading removal, bullet normalization, and repeated-punctuation collapse. Two .docx modes: extract-only (all cleaners) or clean-in-place (preserves tables, images, hyperlinks, headers/footers, styles, track changes).

أصلح التحولات الشائعة من النص المنسوخ — خاصةً من ملفات PDF والصفحات وOCR ومعالجات النصوص. الافتراضي (12 عملية): إصلاح mojibake، توسيع الرابطات، تطبيع علامات الاقتباس/الشرطات، إزالة أحرف العرض الصفري/التحكم، وصل الكلمات المقسمة، إصلاح الاستشهادات المعطوبة، إزالة أرقام الصفحات، وطي المسافات. الصارم (24 عملية، زر ε): يضيف إزالة BOM، CRLF→LF، تحويل المسافات غير القابلة للكسر، تطبيع مسافات Unicode، إزالة الواصلات الناعمة، إزالة محددات التغيير، تحويل علامات الحذف، إزالة النجوم، إزالة عناوين Markdown، تطبيع النقاط، وطي علامات الترقيم المكررة. وضعان لـ .docx: استخراج فقط (كل المنظفات) أو التنظيف في الموضع (يحفظ الجداول والصور والروابط والترويسات/التذييلات والأنماط وتتبع التغييرات).

Citation Manager — Validate Against BibTeX

إدارة الاستشهادات — التحقق من BibTeX

Three checks: (1) Undefined citations — every (Author, Year) in your draft that doesn't match a .bib entry (the citations most likely to be fabricated or wrong). (2) Unused references — every .bib entry never cited in your draft, helping you trim your reference list before submission. (3) Citation count per reference — spot references cited only once (often a token citation). All parsing is local. No .bib content leaves your device. The BibTeX parser is hand-written (no third-party BibTeX dependency).

ثلاثة فحوصات: (1) الاستشهادات غير المعرفة — كل (المؤلف، السنة) في مسودتك التي لا تطابق إدخالاً في .bib (الاستشهادات الأكثر احتمالاً لأن تكون مختلقة أو خاطئة). (2) المراجع غير المستخدمة — كل إدخال في .bib لم يُستشهد به في مسودتك، مما يساعدك على تقليم قائمة المراجع قبل الإرسال. (3) عدد الاستشهادات لكل مرجع — رصد المراجع المستشهد بها مرة واحدة فقط (غالباً استشهاد شكلي). كل التحليلات محلية. لا يغادر أي محتوى .bib جهازك. محلل BibTeX مكتوب يدوياً (بلا تبعية BibTeX خارجية).

Document Statistics & Structure Analyzer

إحصاءات الوثيقة ومحلل البنية

Document Statistics: word/sentence/paragraph/section counts, citation count, average sentence length, type-token ratio, complex-word ratio, estimated reading time (200 wpm), Flesch Reading Ease, Flesch-Kincaid Grade Level, Gunning Fog Index — with a comparison panel against common journal targets (Nature ~5,000 words; ICMJE medical ~3,500 words; IEEE conference ~6,000 words). Structure Analyzer: extract the heading tree, get suggestions for missing sections (Introduction, Methods, Results, Discussion, Conclusion), spot short sections.

إحصاءات الوثيقة: أعداد الكلمات/الجمل/الفقرات/الأقسام، عدد الاستشهادات، متوسط طول الجملة، نسبة التنوع اللفظي، ونسبة الكلمات المعقدة، ووقت القراءة المقدّر (200 كلمة/دقيقة)، وسهولة القراءة Flesch، مستوى Flesch-Kincaid، مؤشر Gunning Fog — مع لوحة مقارنة مقابل أهداف المجلات الشائعة (Nature ~5,000 كلمة؛ ICMJE الطبية ~3,500 كلمة؛ مؤتمرات IEEE ~6,000 كلمة). محلل البنية: استخراج شجرة العناوين، اقتراحات للأقسام المفقودة (المقدمة، المنهجية، النتائج، المناقشة، الخاتمة)، رصد الأقسام القصيرة.

Style Analysis & Voice Consistency

تحليل الأسلوب واتساق صوت الكاتب

Compare a draft's stylistic profile to a sample of your own prior writing. Reports 12 metrics: sentence length (mean + SD), vocabulary diversity (type-token ratio), passive-voice density, hedging density, connector density, first-person singular/plural ratio, citation density, Flesch/Flesch-Kincaid/Gunning Fog, and complex-word ratio. Voice Consistency flags within-document stylistic inconsistencies where sentence length, hedging, or vocabulary abruptly shifts — so you can spot drafts that drift away from your usual register. This does not predict or attempt to lower AI-detector scores.

قارن الملف الشخصي الأسلوبي لمسودتك بعينة من كتاباتك السابقة. يُقدّم 12 مقياساً: طول الجملة (المتوسط + الانحراف المعياري)، تنوع المفردات (نسبة التنوع المعجمي)، كثافة المبني للمجهول، كثافة التردد، كثافة الروابط، نسبة المتكلم المفرد/الجمع، كثافة الاستشهادات، Flesch/Flesch-Kincaid/Gunning Fog، ونسبة الكلمات المعقدة. يرصد اتساق الصوت التباينات الأسلوبية داخل الوثيقة حيث يتغير طول الجملة أو التردد أو المفردات فجأة — لترصد المسودات التي تنحرف عن أسلوبك المعتاد. لا يتنبأ هذا ولا يحاول خفض درجات كواشف الذكاء الاصطناعي.

Abstract Generator & Disclosure Assistant

مولّد الملخصات ومساعد الإفصاح

Abstract Generator: a local LLM produces a Background/Methods/Results/Conclusions abstract from your draft, plus section-by-section commentary. Disclosure Assistant: generate venue-compliant AI-use disclosure statements for ICMJE (JAMA, NEJM, Lancet, BMJ), Nature Portfolio, IEEE (all societies), Elsevier (2,500+ journals), ACL/EMNLP/NAACL, or a generic/custom venue. Each template includes a link to the venue's official AI-use policy.

مولّد الملخصات: ينتج نموذج لغوي محلي ملخصاً مهيكلاً (الخلفية/المنهجية/النتائج/الاستنتاجات) من مسودتك، بالإضافة إلى تعليقات قسم بقسم. مساعد الإفصاح: قم بإنشاء بيانات إفصاح متوافقة مع معايير إستخدام الذكاء الاصطناعي لـ ICMJE (JAMA, NEJM, Lancet, BMJ)، Nature Portfolio، IEEE (كل الجمعيات)، Elsevier (2,500+ مجلة)، ACL/EMNLP/NAACL، أو جهة عامة/مخصصة. يتضمن كل قالب رابطاً إلى سياسة استخدام الذكاء الاصطناعي الرسمية للجهة.

Writing Journal & Appeal Letter

مذكرات الكتابة وخطاب الطعن

Writing Journal: auto-saves timestamped snapshots of your draft, creating a verifiable process record for authentic-authorship evidence. Appeal Letter: if falsely flagged by an AI detector, generate a professional, evidence-based appeal letter citing the peer-reviewed literature (Liang et al. 2023; Weber-Wulff et al. 2023; Laban et al. 2024). The Authenticity Risk Profiler surfaces whether your draft's perplexity/burstiness proxies overlap with typical AI-text profiles — not an evasion tool, an awareness tool.

مذكرات الكتابة: تحفظ تلقائياً لقطات مؤطرة زمنياً لمسودتك، منشئةً سجل عملية قابل للتحقق كدليل على الأصالة. خطاب الطعن: إذا تم تصنيفك زوراً بواسطة كاشف ذكاء اصطناعي، أنشئ خطاب طعن مهني قائم على الأدلة يستشهد بالأدبيات المحكمة (Liang et al. 2023؛ Weber-Wulff et al. 2023؛ Laban et al. 2024). يُظهر مُلف مخاطر الأصالة ما إذا كانت مقاييس الحيرة/الانفجار في مسودتك تتداخل مع أنماط نصوص الذكاء الاصطناعي النموذجية — ليس أداة تهرب، بل أداة وعي.

Privacy Audit — Verify Every Call Yourself

تدقيق الخصوصية — تحقق من كل اتصال بنفسك

An in-session log of every file read and outbound HTTP call. The summary card shows total events, file reads, HTTP calls, Ollama commands, bytes in/out, and unique outbound hosts contacted — which should only ever show registry.ollama.ai (model downloads). Any other host is a red flag. Filterable event table with timestamps. The audit log is in-memory only — cleared on app close — because persisting it would create a record of every file you read, which is the opposite of privacy.

سجل داخل الجلسة لكل ملف يُقرأ وكل اتصال HTTP خارجي. تُظهر بطاقة الملخص إجمالي الأحداث، قراءات الملفات، اتصالات HTTP، أوامر Ollama، البايتات الداخلة/الخارجة، والمضيفين الخارجيين الفريدين — الذين يجب أن يظهروا فقط registry.ollama.ai (تنزيل النماذج). أي مضيف آخر إشارة خطر. جدول أحداث قابل للتصفية مع طوابع زمنية. سجل التدقيق في الذاكرة فقط — يُمحى عند إغلاق التطبيق — لأن حفظه سيُنشئ سجلاً بكل ملف قرأته، وهو عكس الخصوصية.

Local Chat & Saved Work

المحادثة المحلية والعمل المحفوظ

Chat: local-only chat with any installed model. The system prompt includes a guardrail that refuses requests to evade AI detectors or fabricate citations. Use it for brainstorming phrasing, asking for critique of a paragraph, generating outlines, or sanity-checking an argument. Saved Work: opt-in local persistence (disabled by default). When enabled, drafts, chat transcripts, and disclosure statements save as plain JSON files in %APPDATA%\com.scholarscribe.app\data\ — inspectable in any text editor, never synced to cloud, with per-draft delete and "Delete all" options.

المحادثة: محادثة محلية فقط مع أي نموذج مثبّت. تتضمن رسالة النظام حاجزاً يرفض طلبات التهرب من كواشف الذكاء الاصطناعي أو اختلاق الاستشهادات. استخدمها لعصف العبارات، أو طلب نقد فقرة، أو توليد المخططات، أو التحقق من حجة. العمل المحفوظ: حفظ محلي اختياري (معطّل افتراضياً). عند التفعيل، تُحفظ المسودات ومحاضر المحادثات وبيانات الإفصاح كملفات JSON صريحة في %APPDATA%\com.scholarscribe.app\data\ — قابلة للفحص في أي محرر نصوص، لا تُزامن مع السحابة أبداً، مع خيارات حذف لكل مسودة و"حذف الكل".

  • Helps you draft, paraphrase, and refine your own writing with a local LLM.
  • يساعدك على صياغة وإعادة كتابة وتحسين كتاباتك الخاصة باستخدام نموذج لغوي محلي.
  • Compares a draft's stylistic profile to a sample of your own prior writing so you can decide whether the draft still sounds like you.
  • يقارن الملف الأسلوبي لمسودتك بعينة من كتاباتك السابقة لتقرر إن كانت المسودة لا تزال تشبهك.
  • Generates disclosure statements so you can comply with journal and conference AI-use policies.
  • يُنشئ بيانات إفصاح لتلتزم بسياسات استخدام الذكاء الاصطناعي للناشرين.
  • Educates you about how AI-detection tools work and what their known limitations are.
  • يُعلّمك كيف تعمل كواشف الذكاء الاصطناعي وما حدودها المعروفة.
  • Validates your citations against your .bib file — reducing the risk of fabricated references.
  • يتحقق من استشهاداتك مقابل ملف .bib — مما يقلل من خطر المراجع المختلقة.
  • Maintains a timestamped draft history as verifiable evidence of authentic authorship.
  • يحفظ تاريخاً مؤطراً زمنياً للمسودات كدليل قابل للتحقق على الأصالة.
  • Generates evidence-based appeal letters for researchers falsely flagged by AI detectors.
  • يُنشئ خطابات طعن قائمة على الأدلة للباحثين المُصنَّفين زوراً بواسطة كواشف الذكاء الاصطناعي.
  • Cleans text artifacts from copy-pasted content without rewriting it.
  • ينظّف تحولات النص المنسوخ دون إعادة كتابته.

What it explicitly does NOT do

ما لا يفعله صراحةً

  • Does not target or attempt to lower AI-detection scores. No "marker-targeting" engine. No Turnitin/GPTZero/Originality score-reduction pipeline. No stealth modes.
  • لا يستهدف ولا يحاول خفض درجات كشف الذكاء الاصطناعي. بلا محرك استهداف علامات. بلا خط أنابيب لخفض درجات Turnitin/GPTZero/Originality. بلا أوضاع تخفّي.
  • Does not help misrepresent AI-generated text as original human work. The chat module's system prompt explicitly refuses requests to evade detectors or submit AI output as one's own.
  • لا يساعد في تمرير النص المولّد بالذكاء الاصطناعي كعمل بشري أصيل. ترفض رسالة نظام المحادثة صراحةً طلبات التهرب من الكواشف أو تقديم مخرجات الذكاء الاصطناعي كعمل ذاتي.
  • Does not contact any third-party API. The only network call is to registry.ollama.ai when you choose to download a model — and that call carries no text, no prompts, no usage data.
  • لا يتصل بأي واجهة طرف ثالث. الاتصال الشبكي الوحيد هو إلى registry.ollama.ai عند تنزيل نموذج — ولا يحمل أي نص أو استفسارات أو بيانات استخدام.
  • Does not collect telemetry, analytics, or crash reports.
  • لا يجمع بيانات استخدام أو تحليلات أو تقارير أعطال.
  • Does not read any file you didn't explicitly pick in a file dialog.
  • لا يقرأ أي ملف لم تختره صراحةً في حوار الملفات.
  • Does not fabricate citations. The Chat tab's system prompt forbids this; the Citation Manager catches accidental fabrication.
  • لا يختلق استشهادات. تمنع رسالة نظام المحادثة ذلك؛ وتلتقط إدارة الاستشهادات الاختراق العرضي.

Getting Started in 5 Minutes

البدء في 5 دقائق

1

Install Ollama

ثبّت Ollama

Download Ollama from ollama.com/download (Windows installer, ~150 MB). Run the installer — Ollama starts automatically as a background service. ScholarScribe talks to it on http://127.0.0.1:11434.

نزّل Ollama من ollama.com/download (مثبّت ويندوز، ~150 ميجابايت). شغّل المثبّت — يبدأ Ollama تلقائياً كخدمة خلفية. يتواصل سكولار سكرايب معه على http://127.0.0.1:11434.

2

Install ScholarScribe

ثبّت سكولار سكرايب

Grab the latest .msi or .exe from the Releases page. Double-click to install. ScholarScribe will appear in your Start menu. (Or build from source: Rust 1.77+, Node.js 18+, Tauri prerequisites — npm install & npm run tauri build.)

احصل على أحدث .msi أو .exe من صفحة الإصدارات. اضغط مرتين للتثبيت. سيظهر سكولار سكرايب في قائمة ابدأ. (أو ابنِ من المصدر: Rust 1.77+، Node.js 18+، متطلبات Tauri — npm install وnpm run tauri build.)

3

Install a Model & Write

ثبّت نموذجاً واكتب

Open the Models tab and install a model that fits your RAM (e.g. Gemma 3 4B for 8 GB RAM, Qwen 3 14B for 16 GB). Then load your .docx or .bib file in the relevant tab. Your text never leaves your machine.

افتح تبويب النماذج وثبّت نموذجاً يناسب رامك (مثلاً Gemma 3 4B لـ 8 جيجا رام، Qwen 3 14B لـ 16 جيجا). ثم حمّل ملف .docx أو .bib في التبويب المناسب. لا يغادر نصك جهازك أبداً.
